Inline-elements list

This list is intended to be an exhaustive list of all current inline elements that are not deprecated as of this page's last update. Do keep in mind that "inline elements" is somewhat arbitrary, as any element can be set to display: inline;, but this list assumes an inline element to be any element that is generally set to display: inline; by default by most major browsers, including Firefox, Chrome, Safari, and more.

Practically speaking, an inline element is anything that is expected to be used in within blocks of text (with exceptions like <img>, which are inline despite not usually being found in the middle of blocks of text). For example, <span> is inline as the intended purpose of it is for demarcating certain sections of text. <div> is not inline, not only for reasons listed above, but also because it would be impractical and counterintuitive to do so.

  • <a>

    The <a> element is referred to as an "anchor" or as "anchor tags". It serves the function of demarcating text to be a hyperlink, either leading to another page on its own website or externally to another website.

    Pipertext rules!

  • <abbr>

    The <abbr> element denotes an abbreviation. Making use of the title attribute allows for the containing text to offer a tooltip when hovered over.

    My favorite video game is TF2!

  • <address>

    The <address> element denotes an address. The kind of address can vary, anything from physical addresses to phone numbers and emails. Think of it more as a pointer to somebody, somewhere, or something. When denoting something like an email, this element will usually contain an <a> element with an attribute of href='mailto:contact@pipertext.net'.

    You can always contact me at

    contact@pipertext.net
    !

  • <audio>

    The <audio> element inserts an audio interface, usually for pausing, changing the volume, and skipping around an audio clip with the use of the controls attribute.

  • <b>

    <b> is the boldface element. Nowadays, it's often called the "Bring Attention To" element in order to distinguish it from <strong> tags. While both generally have the same visual effect, <b> should only be used to bring the reader's eyes/attention to certain word(s), not for granting importance to words.

    I love writing HTML!

  • <bdi>

    The <bdi> element stands for "bidirectional isolate". It's used as a means to tell the browser to treat the contained text as separate from the surrounding text with regards to the direction in which the text flows. That sounds confusing, but I assure you it isn't; for example, Arabic is written right-to-left. If you had some Arabic in the middle of an English paragraph, you can wrap that Arabic in the <bdi> element to ensure it is written in the right-to-left format.

    Howdy there كليتوس!

  • <bdo>

    The <bdo> element can be used to force text into either right-to-left (dir='rtl') or left-to-right (dir='ltr') directionalities.

    I'm kind of hard to read.

  • <br>

    Perhaps the most interesting case of inline element, <br> produces a line break. Despite the fact that it basically coerces text into a one-time block-level structure, it is an inline element and can only be made use of as such. <br> is a void element.

    This is the first line of text.
    And this is the second line of text, directly following <br>.
